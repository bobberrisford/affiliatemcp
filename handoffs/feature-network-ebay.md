# Handoff — `feature/network-ebay`

**Chunk**: eBay Partner Network adapter (fifth bundled network, peer to
Awin / CJ / Impact / Rakuten)
**Branch**: `feature/network-ebay`
**Base**: tip of `claude/affiliate-mcp-orchestration-qfKw4` (commit
`19d9bc3 — Structural build complete (chunks 1-12 merged); contribution test +
launch prep next`)

## What I did

Implemented an `affiliate-mcp` adapter for eBay Partner Network (EPN) under
`src/networks/ebay/`, mirroring the layout of the four existing adapters.
Wired it into the network aggregator. Added unit tests and fixtures, a
findings doc and a setup doc, env-example entries, and updated the two
integration tests that asserted the registry contained exactly four
adapters.

**The adapter was implemented from public documentation only.** No real EPN
credentials were available in the environment and no live API calls were
made. Fixtures are synthesised from the documented response shapes. Claim
status ships as `experimental`.

### Files added

- `src/networks/ebay/network.json` — canonical manifest (oauth2,
  3 env vars, approval-gated, `claim_status: experimental`,
  `adapter_version: 0.1.0`, three known limitations).
- `src/networks/ebay/auth.ts` — OAuth2 client-credentials flow with
  in-memory token cache (the only module-level mutable state, mirrors
  Rakuten's precedent). Exports `getAccessToken`, `verifyAuth`,
  `validateCredential`, `_resetTokenCache`. Honours
  `EBAY_TOKEN_URL` / `EBAY_OAUTH_SCOPE` overrides for the sandbox / test
  seam.
- `src/networks/ebay/client.ts` — single sanctioned `fetch` path,
  acquires a token inside the resilience wrapper, sends
  `X-EBAY-C-MARKETPLACE-ID` on every request. Honours `EBAY_BASE_URL`
  override.
- `src/networks/ebay/setup.ts` — three `SetupStep` records for the
  wizard. Step 1's description names the Partner Network enrolment
  approval and its typical 1-3 day wait.
- `src/networks/ebay/adapter.ts` — the `NetworkAdapter` implementation
  with a heavily-commented file header explaining the EPN "one
  advertiser, programme==campaign" model, the seven publisher
  operations, the status mapping
  (`PENDING`/`CLEARED`/`PAID`/`CANCELLED` → canonical), and the
  cardinal rules.
- `tests/networks/ebay/adapter.test.ts` — 37 tests covering
  transformers, list/get programmes, listTransactions (chunking,
  filters, §15.9 unpaid-age, §15.10 reversed-reason),
  listClicks (programme filter), generateTrackingLink
  (deterministic, URL-encoded, rotation override, no-fetch),
  verifyAuth (200/401/missing-cred), validateCredential (all three
  fields + unknown field), token caching, §15.4 error transparency
  (500 body, 401→auth_error), and capabilitiesCheck.
- `tests/networks/ebay/manifest.test.ts` — schema conformance.
- `tests/fixtures/ebay/{campaigns,transactions,clicks,token}.json` —
  synthesised plausible response shapes.
- `docs/findings/ebay.md` — the editorial findings doc (what worked,
  what didn't, the cardinal shape difference, token longevity, future

- `docs/networks/ebay.md` — per-network setup walkthrough with
  screenshot placeholders, prerequisites, common-failures section,

## What's unfinished

- **No live verification.** This is the principal caveat. The
  adapter has not been exercised against a real EPN account. Promote
  `claim_status` from `experimental` → `partial` after a first
  smoke test and to `production` after a live acceptance test (per
  `docs/network-claim-process.md`).
- **`EBAY_CAMPAIGN_ID` is not yet derived.** The user must paste the
  numeric campaign ID from the EPN dashboard. If
  `/affiliate/campaign/v1/campaign` turns out to be available to
  standard publisher credentials at runtime, move to the
  `derivedValues` pattern (offer the first active campaign as the
  default).
- **`X-EBAY-C-MARKETPLACE-ID` defaults to `EBAY_GB`.** UK English
  project, UK reporting default. US publishers must set
  `EBAY_MARKETPLACE_ID=EBAY_US`. Worth flagging in the wizard's
  post-success summary in a future iteration.
- **README editorial copy still says "four bundled adapters".** I
  regenerated the README table block (the auto-generated region
  between the marker comments) and the REPORT.md, but did not
  rewrite the surrounding prose ("With four networks bundled — Awin,
  CJ Affiliate, Impact, and Rakuten Advertising") or the
  Acknowledgements section. AGENTS.md is explicit about not
  modifying another network's adapter; by the same restraint I
  treated the broader README copy as a coordinated edit for a
  separate "promote eBay across the docs" pass once the adapter is
  out of `experimental`. The auto-generated table is the
  load-bearing piece and it does list eBay.
- **`src/skills/affiliate-network-setup-help/SKILL.md`** has a line
  ("for all four networks supported here") that will also need a
  follow-up edit once eBay is promoted.
- **No screenshots.** The setup doc uses `[SCREENSHOT: …]`
  placeholders pointing into `docs/networks/images/ebay/` — there
  is no `images/ebay/` directory yet. Matches the precedent of the
  Awin setup doc which has the same placeholders without images at
  this stage of the build.

## What surprised me

- **EPN is structurally unlike the other four networks.** There is
  only one advertiser (eBay itself); the concept that corresponds
  to "a programme" on every other network is an EPN **campaign**
  (a publisher-owned tracking bucket). I considered mapping eBay
  marketplaces to programmes instead, but the campaign model is the
  honest one — it is what EPN's reporting endpoints actually
  partition on, and `programmeId` is exactly the value the user
  passes as `campid` on the rover URL. I called this out in the
  network.json `known_limitations`, the adapter's file-level
  comment, the setup doc, and the findings doc, so a downstream
  skill author who naively assumes "more programmes = more revenue"
  has a fighting chance of reading the caveat.

- **The token exchange is the auth check.** Most networks I worked
  with want a separate `/me` or `/publishers` call after the
  token exchange to confirm identity. eBay's OAuth endpoint refuses
  bad credentials cleanly with a 401 (and a JSON body naming the
  bad field), so the exchange itself proves the credentials are
  valid. This kept `verifyAuth` cheap — one POST, one parse, done
  — and matched the wizard's "fast validation between prompts"
  ergonomics.

- **EPN exposes click data.** Unlike Awin and CJ, EPN has a
  `/affiliate/reporting/v1/click` endpoint. `listClicks` is a real
  operation rather than a `NotImplementedError`. This lifts EPN's
  supported-ops count to 7/7 in the auto-generated README table,
  which is correct but slightly misleading because the click
  endpoint is reporting-delayed; the README's terse "see notes"
  column points the reader at the findings doc for the caveat.

- **The dual interpretation of `programmeId` in
  `generateTrackingLink`.** EPN's rover URL needs a campaign ID
  (`campid`) and a destination URL (`mpre`). I made the
  adapter accept either an EPN campaign ID OR an item ID as
  `programmeId` because callers often have the latter handy (a
  blog author thinking "I want to link to listing 38512…"). The
  campaign ID is then drawn from `EBAY_CAMPAIGN_ID`. I documented
  the dual interpretation in the adapter comment and the setup
  doc; if it turns out to be confusing in practice, narrowing it
  to "campaign ID only" is a one-line change.

- **`X-EBAY-C-MARKETPLACE-ID` is sticky in mysterious ways.** Some
  eBay endpoints require it, some ignore it, some 400 without it.
  I send it on every call to remove a class of "missing header"
  debugging from the future.

## Recommended next steps

1. **Live verification.** Get a real EPN publisher account, run
   `npx affiliate-mcp setup ebay`, and exercise each of the seven
   tools. Most likely fix-ups: pagination shape on the reporting
   endpoints, edge-case status strings, marketplace header
   requirements per endpoint. Promote to `partial` once green.

2. **`derivedValues` for `EBAY_CAMPAIGN_ID`.** If the
   `/affiliate/campaign/v1/campaign` listing endpoint is reachable
   with the standard publisher credentials, add a `derivedValues`
   implementation that offers the first active campaign as the
   wizard's default (mirrors Awin's `AWIN_PUBLISHER_ID`
   derivation).

3. **Promote eBay across the editorial docs.** Once the adapter
   leaves `experimental`, update:
   - The README prose lines that say "four networks bundled".
   - `src/skills/affiliate-network-setup-help/SKILL.md`.
   - The Acknowledgements section of README.md.
   - Any handoff or governance doc that lists the four bundled
     networks by name.

4. **Screenshots for the setup doc.** Add
   `docs/networks/images/ebay/{1..5}-*.png` matching the
   placeholders. Same pattern as Awin.

5. **Subid / `customid` support.** Widening the canonical
   `generateTrackingLink` input shape across all networks to take
   an optional `subId` would let EPN populate `customid` on the
   rover URL. Touches the shared type contract, so requires its own
   PR + discussion.

6. **Sandbox testing.** With `EBAY_BASE_URL=https://api.sandbox.ebay.com`
   and `EBAY_TOKEN_URL=https://api.sandbox.ebay.com/identity/v1/oauth2/token`,
   the adapter should drive cleanly against eBay's sandbox. Use
   this for CI-friendly continuous validation once a sandbox EPN
   app exists.
