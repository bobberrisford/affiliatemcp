### 4. Comparison-table PNG — failed to render in this sandbox

`npm run generate:report-image` was attempted. The script exits
cleanly with the "Playwright is not installed" message when the
module is absent. After `npm install --save-dev playwright`,
`npx playwright install chromium` failed with HTTP 403 from
`cdn.playwright.dev` ("Host not in allowlist") — the Playwright
browser-binary CDN is blocked by the sandbox's network policy.

The npm package itself installed cleanly (3 packages, audited 300,
no breakage); the rasterisation step is what cannot complete. Per
the chunk brief, I uninstalled Playwright again rather than leaving
a dev-dependency that does not work in this environment — the
`package.json` and `package-lock.json` are back to their pre-chunk
state.

**Outcome**: PNG NOT rendered. Documented in
`docs/launch/CHECKLIST.md` section 6 with the exact reproduction
steps for the maintainer to run on a machine with Internet access.
The HTML composition function in `scripts/generate-report-image.ts`
is pure and was unaffected.

## What surprised me

- **The README generator is well-behaved.** I half-expected
  `npm run generate:readme` to re-scan the body and re-introduce
  the "four networks" prose. It does not — it only touches the
  table region between the markers. The Chunk-7 design held.
- **The Playwright failure mode is structural, not transient.**
  The script's "Playwright not installed" path is well-written and
  the rasterisation path also writes the HTML alongside on
  failure as a debugging aid. Neither helps when the sandbox
  blocks the browser CDN; that is purely an environment issue.
  Documenting it in the checklist is the right move.
- **The eBay handoff already documented this work.** The
  `Recommended next steps` block in `handoffs/feature-network-ebay.md`
  named exactly the README prose correction this chunk completes,
  plus the follow-ups (AGENTS.md, the setup-help skill) that this
  chunk deliberately scoped out. The continuity is intentional.
