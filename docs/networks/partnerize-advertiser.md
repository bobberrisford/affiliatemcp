# Partnerize (Advertiser) — setup guide

This adapter integrates with the **Partnerize Brand API** (v3). It acts on behalf
of a brand (advertiser) account, enumerating campaigns, publishers, conversions, and
performance analytics. One set of credentials may address multiple campaigns.

---

## Prerequisites

- A Partnerize brand account with API access enabled.
- Access to the Partnerize dashboard at [https://console.partnerize.com](https://console.partnerize.com).
- No manual approval step is required beyond the standard Partnerize brand account sign-up.

---

## Credentials needed

Two credentials are required. Both are found on the same dashboard page.

### `PARTNERIZE_APPLICATION_KEY`

The Application Key identifies which Partnerize network your account belongs to.

**Where to find it:**
1. Sign in at [https://console.partnerize.com](https://console.partnerize.com).
2. Navigate to **Settings** → **API Credentials**.
3. Copy the value shown next to **Application Key**.

All users on the same Partnerize network share the same Application Key.

### `PARTNERIZE_USER_API_KEY`

The User API Key authenticates you (the individual user) against the API.

**Where to find it:**
1. Sign in at [https://console.partnerize.com](https://console.partnerize.com).
2. Navigate to **Settings** → **API Credentials**.
3. Copy the value shown next to **User API Key**.

This key is per-user. Keep it confidential; anyone with this key can read your
campaign data.

---

## Setup steps

1. Sign in at [https://console.partnerize.com](https://console.partnerize.com).
2. Navigate to **Settings** → **API Credentials**.
3. Run the setup wizard:
   ```
   affiliate-networks-mcp setup partnerize-advertiser
   ```
4. Paste the **Application Key** when prompted.
5. Paste the **User API Key** when prompted. The wizard will probe the Brand API
   to confirm the credentials are valid.
6. The wizard will discover which campaigns (brands) are accessible. Map each
   campaign to a logical brand slug in `brands.json` using the displayed campaign IDs.

---

## Common failures

1. **HTTP 401 — credentials rejected.**
   The Application Key or User API Key is incorrect or was pasted with
   leading/trailing whitespace. Go to **Settings** → **API Credentials** and
   copy the values again without modification.

2. **Empty campaign list.**
   The authenticated user account does not have access to any active campaigns.
   Confirm the user account has been assigned to at least one campaign in the
   Partnerize dashboard under **Campaigns** → **Campaign Access**.

3. **`config_error`: networkBrandId missing.**
   Advertiser-side tools require a `brand` argument that the tool dispatcher
   resolves to a `campaign_id` via `brands.json`. Run
   `affiliate-networks-mcp setup partnerize-advertiser` to complete the
   brand-discovery sub-flow and populate `brands.json`.

---

## Known limitations

- **Not verified against a live account.** This adapter was built from public
  API documentation. Endpoint shapes and field names are marked `// TODO(verify)`
  and should be confirmed with a real Partnerize brand account before relying on
  the output in production.
- **Click-level data is not exposed by the Brand API.** The `listClicks`
  operation is unsupported. Use `getProgrammePerformance` for aggregated click
  counts per publisher.
- **`getProgramme` is not implemented at v0.1.** Use `listProgrammes` to list
  campaigns and filter client-side.
- **`getEarningsSummary` is not implemented at v0.1.** Use
  `getProgrammePerformance` for the per-publisher commission rollup.
- **`generateTrackingLink` is publisher-side.** Tracking links are generated by
  publishers, not brands; this operation is not available on the advertiser adapter.

## Verifying

```
affiliate-networks-mcp test partnerize-advertiser
```

This runs the live diagnostic suite. The same engine runs during
`npm run validate:network -- partnerize-advertiser`. Both must pass before the
adapter is considered verified.
