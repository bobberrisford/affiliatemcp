## What I did

### Generators (`scripts/`)
- **`scripts/report-data.ts`** — shared data-loading module used by all three
  generators. Reads every `src/networks/<slug>/network.json` and matching
  `docs/findings/<slug>.md`. Sorts networks alphabetically by slug for
  deterministic output. Pure helpers for formatting setup-time, approval cell,
  notes summary, supported-op count, and per-operation flags. Live-credentials
  injection is supported (the caller can pass capabilities) but the CLI does
  not currently invoke `runDiagnostic()` at generation time — deferred with a
  comment in `generate-report.ts` because the diagnostic engine would spam
  auth failures when no creds are configured. The infrastructure is in place;
  flipping it on is a one-line change in the CLI once Chunk 8's credential
  setup is exercised.
- **`scripts/generate-report.ts`** — composes `REPORT.md`. Sections in PRD
  order: heading + tagline, methodology, summary table, per-network section
  (quick facts → operations table → known limitations → findings prose
  embedded verbatim), closing with reproduce instructions. Pure
  `renderReport(data, options)` for testing; the CLI is a thin wrapper that
  writes the result to `REPORT.md` at the repo root.
- **`scripts/generate-readme-table.ts`** — updates the network table inside
  the marked block in `README.md` (markers `AFFILIATE_MCP_NETWORK_TABLE_START`
  / `..._END`). Inserts the block at the end of the file if the markers are
  absent. Idempotent.
- **`scripts/generate-report-image.ts`** — pure HTML composition + Playwright
  rasterisation to `docs/images/report-table.png`. Playwright is loaded via
  dynamic `import('playwright')` so the script can run even when the dep is
  not installed; in that case it prints a clear install message and exits 1
  (no silent no-op, no fake image). Page width 1200px, system font, light
  background.

## What's unfinished

- **Live diagnostic injection at generation time** — the data-loader supports
  it but the `generate-report.ts` CLI does not yet call `runDiagnostic()`.
  Reason: with no credentials configured (the default state pre-launch),
  the diagnostic would emit a flurry of `verifyAuth` failures into stderr.
  When Chunk 8 wires credential setup, flip the switch by uncommenting the
  block in `runCli()` (or by adding an `--include-live` flag).
- **Playwright** is not installed as a devDependency. The script handles
  this gracefully (prints an install hint, exits 1). I chose not to install
  it because the binary download is heavy and the chunk brief explicitly
  permits skipping the install with a note in the handoff. To enable the
  image generator:
  ```
  npm install --save-dev playwright
  npx playwright install chromium
  npm run generate:report-image
  ```
  The HTML composition is already test-covered, so the only risk surface
  added by enabling it is the rasterisation call itself.
- **`docs/benchmark-methodology.md`** — referenced from the methodology
  section of REPORT.md, but not yet written. The placeholder paragraph
  in the report tells the reader it's coming. Chunk 12.
- **`docs/networks/<slug>.md`** — README links to these per-network setup
  pages. Chunk 8 ships them.
- **`CONTRIBUTING.md`** — README links to it; not yet written. Chunk 11.

## What surprised me

- **Findings docs contain `# H1` headings.** Embedding them verbatim under
  the per-network `### Findings` subsection produces a structurally unusual
  document with multiple H1s after the report's own H1. The chunk brief is
  explicit: "pulled VERBATIM ... keep formatting". I kept the originals
  intact rather than rewriting the headings. If the launch demo wants a
  cleaner H-hierarchy, the post-processing step is a single regex pass in
  `renderNetworkSection` to demote `^#` lines inside the findings. Not done
  by default because demoting headings is the kind of "paraphrase" the brief
  explicitly forbids.
- **Rakuten findings doc contains the literal string "Rakuten is bad"** —
  inside a quoted disclaimer ("describe what is true, not 'Rakuten is bad'").
  The string survives the verbatim embed and shows up in `REPORT.md` line
  659. The tone-check test runs against fixtures so it does not trip on
  this. The actual sentence reads as intended (it's a meta-comment by the
  author, not a verdict). Worth a quick eyeball during launch review; if
  the project wants the report itself to be perfectly clean of snarky
  vocabulary even quoted, the simplest fix is for Rakuten's findings doc
  to be edited (out of scope for this chunk per the "do not edit findings"
  rule, but flagging for follow-up).
- **`tsconfig.json` `rootDir: src`** excludes `scripts/` from the build, so
  the new generator code doesn't end up in `dist/`. That's the right
  outcome — these are developer scripts, not server code — and the
  existing `tsconfig.dev.json` already includes `scripts/` for type-checking.
  No tsconfig changes needed.
- **`process.argv[1]?.endsWith('generate-report.ts')`** is the existing
  is-main heuristic in `validate-network-json.ts`. I kept the same pattern
  in the new scripts for consistency; it survives the `tsx` shim and gives
  the test importers a clean no-op when the module is imported rather than
  invoked.
